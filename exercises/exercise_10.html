Exercise 10 - Ray marching SDFs

Learning objectives

In this exercise we will practice some simple post-processing effects. We will be learning how to:

  • Add SDF for geometric shapes.
  • Create complex scenes combining SDFs.

Initial implementation

We have created a small ray marcher for you to play with.

This raymarcher can render objects with special shaders. By default it renders a single full screen quad, and that might be enough for this exercise. You can easily add new objects if you consider that necessary.

Most of the work in this exercise should be modifying your distance function in the fragment shader.

To make your life easier, you can reload shaders by just pressing the R key. It doesn't work well yet if you are adding new uniforms, but if you are just tweaking values in your distance function it can save some time. If something breaks, just restart the application.

Editing SDF shaders.

You will need a new SDFShader for each ray-marched pass you want to add. You should use raymarcher.vert as vertex shader, and a new .frag file as fragment shader, where you will implement your code.

Each .frag file must implement a few things to work properly with the raymarcher:.

  • void GetRayMarcherConfig(out int maxSteps, out float maxDistance, out float surfaceDistance)
    .Provides the configuration values to the raymarcher. I recommend that you use the default values and play with it after you have your scene.
  • struct Output
    Structure that holds all the information that will be used at the end to compute color. You decide what you want to store.
  • void InitOutput(out Output output)
    Initializes the default values for Output.
  • float GetDistance(vec3 p, inout Output output)
    Your main function. This function will return the minimum distance from a point to the entire scene.
    It will be responsible to give values to Output as well.
  • vec4 GetOutputColor(vec3 point, float distance, Output output)
    After the raymarcher finds the correct point, this function has to implement the final color for that pixel, using the information stored in Output.

Adding new SDFs to the library.

You can modify sdflibrary.glsl to add new sdf funcions that will be available in all your SDFShader.

Look at the first link of the references to find some useful ones.

Adding properties.

You can pass uniforms to your fragment shader. They will allow your fragment shader to be dynamic.

  • Declare the uniform in your fragment shader. All uniforms for transforms must be in VIEW space.
  • Set the value as a property in the SDFMaterial of your object.
  • (Optional) Add UI to modify the value of the property in runtime.

(Optional) Adding new objects.

If some of the elements in your scene don't need to interact with others, you can have them in a separate object.

The easiest is to keep everything in the same fullscreen pass object, but spliting the scene can help performance.

To add a new object you have to:

  • Create a new SDFShader with a new .frag shader file.
  • Create a new SDFMaterial that will store the properties of your object.
  • Create or reuse the SDFGeometry of your object. This geometry is only to select which pixels get affected by your shader.
  • Create a new SDFObject with your material and geometry. You may need to set the model matrix for your object, if it is not a fullscreen quad.
  • Add the object to the raymarcher.
Exercise 10.1
 Get familiar with the ray marcher

The ray marcher exercise starts with a default SDFObject. This object is a full screen quad that contains an scene with a blue sphere and a red box.

To start understanding how the raymarcher works, you will have to add controls to move the sphere and the box and change the colors. After that we will apply a new operation that would not be possible with polygons.

Interacting with the scene:

  • In main.cpp: We added some properties to the UI that we would like to use to configure the scene, but we forgot some. Add them to the UI, using the globals already declared.
  • In main.cpp: Set the material properties for the control variables. To make things easier for you, we have left some commented code for the box properties. Implement the missing properties for sphere.
  • In exercise10_1.frag: Replace the constants with uniforms, that will be connected to the material properties.

Now you should be able to move the objects in the scene! Test that you can move, rotate, scale and change color of the objects. Also, spend some time understanding the code in exercise10_1.frag.

If you move the two objects together, you will see that they occlude each other, without using any depth buffer. The trick to achieve this is to keep the minimum of the two distances (sphere and box).

What if we used a different function to combine them? In exercise10_1.frag, try this:

  • Replace min with smin(smooth min). You can open sdflibrary.glsl to read the code to this and other functions.
    smin requires one extra float parameter, k, that you need to add (just hardcode the number there, it is fine).
  • Move the sphere and the box next to each other and try different values of k. You should be able to quickly reload shader code by pressing R key.
  • You may have noticed that the smooth color is all colored with the box color. There is an overloaded version of smin that takes a fourth parameter, blend. This is an out parameter that returns a float between 0 and 1, depending on the closest object. Try this version of the function, and use the blend value to mix the colors, instead of comparing the depth as we initially did.

That feels different, right? If you wanted to do something like that will polygons it would be quite difficult. Explore different values for the parameters and try to understand the code until you feel comfortable with it.
Exercise 10.2 Create your own SDF scene!

This time we are going to do an open exercise so you can unleash your creativity.

Replace the default pass with a fragment shader of your own (you can start with a copy of empty.frag) and modify your distance function to recreate a complex scene.

Some interesting ideas:

  • Try to reproduce some basic scene combining primitives (a famous painting, a cartoon character, or anything you want).
  • Explore animation adding a time uniform to your SDFs.
  • Implement some 3D fractal function like mandelbulb, mengel or julia set.

You should share your results with your classmates by posting them on teams!.
