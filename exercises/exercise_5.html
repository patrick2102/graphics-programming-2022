Exercise 5.1
 Ambient light

Ambient reflection is used as a crude approximation of indirect lighting. To implement this you will need to:

  • Send the following uniforms to the shaders:
    • ambientLightColor: The ambient light color modulated by its intensity.
    • ambientReflectance: How much ambient light the object reflects.
    • reflectionColor: The color of the object.
  • In the vertex shader, following the equations seen in class, compute the ambient reflection and send the final color to the fragment shader
  • In the fragment shader just output the received color

Can you see the car?
Exercise 5.2
Gouraud shading - Diffuse

Diffuse reflection is the light that is scattered and reflected in every direction as it hits the surface of the object. To implement it you will need to:

  • Send the following uniforms to the shaders:
    • light1Position: The position of the first light.
    • light1Color: The first light color modulated by its intensity.
    • diffuseReflectance: How much diffues light the object reflects.
  • In the vertex shader, following the equations seen in class, compute the diffuse reflection, add it to the color from previous steps and send it to the fragment shader
  • In the fragment shader just output the received color

Make sure that all your positions and vectors are represented in the same space! We are computing lighting in world space in this exercise
Exercise 5.3
Gouraud shading - Specular

Specular reflection is the light that gets reflected towards a predominant direction. It creates the spotlights that you can see in the image on the left. To implement it you will need to:

  • Send the following uniforms to the shaders:
    • light1Position: The position of the first light.
    • light1Color: The first light color modulated by its intensity.
    • specularReflectance: How much specular light the object reflects.
    • specularExponent: How concentrated the spotlight is, the higher the value, the smoother is the surface of the material.
  • In the vertex shader, following the equations seen in class, compute the specular reflection, add it to the color from previous steps and send it to the fragment shader
  • In the fragment shader just output the received color

Congratulations! You have completed a Gouraud shading implementation of the Phong reflection model!
Exercise 5.4
Phong shading

Gouraud shading computes lighting per vertex, and interpolate the final color of each pixel using the graphics pipeline. If the mesh has large triangles, this results in the triangular artifacts that you can see in the previous exercise.

Phong shading interpolates the normal of the surface using the graphics pipeline, and computes the final lighting color only at the very end, in the fragment shader

To implement Phong shading you will need to:

  • Copy the lighting computation to fragment shader, from gouraud_shading.vert to phong_shading.frag.
  • You will need all the uniform variables from before on your fragment shader too.
  • You need to send the vertex normal and position from the vertex shader (phong_shading.vert) to the fragment shader using out and in variables. These will get interpolated in the rendering pipeline.

Instead of interpolated shaded values, now you should get a smooth lighting effect.

Congratulations! You have completed a Gouraud shading implementation of the Phong reflection model!

P.S.: Notice how we can now see the reflection of the light on the floor. This was not possible with Gouraud shading, can you tell why this was the case? (Hint: the floor is made up by only two large triangles).
Exercise 5.5
Phong shading - Light attenuation

Lighting loses intensity as the distance between the light emitter and the surface increases. That is because the further the light waves travel, the larger is the area of the spherical surface that it occupies. That means that the lighting area is increasing, but the energy is not (it remains constant)

We can simulate this attenuation by taking the distance between the fragment and the light into account. To implement attenuation:

  • Compute the distance to the light.
  • Compute attenuation as 1 divided by distance squared.

This way of computing attenuation is realistic, but as you can see it is not very convenient to tweak the lighting. In videogames many times we add some linear component as well.

Exercise 5.7
(Optional challenge)		 Additive lights

We have seen a way to render more than one lights. But adding lights to the shader is not always an option. We can't modify the shader every time we want to add a new light. Also, if we want to reuse the same shader, we will be computing extra lights even if they are not present.

Another way to deal with this would be:

  • Modify the shader again to only have one light.
  • Draw the objects normally with the first light.
  • Right after, enable blending and set the factors in glBlendFunc to have additive blending.
  • Change glDepthFunc to GL_LEQUAL.
  • Set the uniforms for the second light, and ambient to 0.
  • Draw the objects again!
  • Disable blending and restore glDepthFunc.

It is not an easy change, but if you do it properly, you will see the same result we had before!
